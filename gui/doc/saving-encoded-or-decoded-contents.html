Saving Encoded Or Decoded Contents

After encoding or decoding your file, you might want to save it.
The application provides you with a method for saving your files.
Saving an encoded content is different from also saving decoded contents.
Let's take a look at both scenarios.

Saving Encoded Contents

After encoding your file, the application will allow you to save the contents.
There are three ways in which you can save encoded contents.

  1. Let Quick Base64 save it with a .txt extension
  2. Provide your own extension and save the file
  3. Copy the file to clipboard

Allowing Quick Base64 to save the file

When you select this option after encoding a file to base64, the application will save your encoded contents in a file with .txt extension.
Also note that, the save file will take the name of the original file just that it will end in a .txt extension

Providing your own extension

When saving your encoded contents, you are allowed to provide your own extension.
To do that, select Save (Give ext.) option and this will allow you to save your file under your preferred extension.
Do note that when you select this option, your file will be saved under a new name not the name of the original file.
The name of the file will be provided by you.

Copying the results to clipboard

Also, you can copy the results into your system's clipboard.

Saving decoded contents

After you are done decoding your file from base64 format, the application provides you with methods to save your file.
Also, there are three ways in which you can save your file.

  • Let Quick Base64 save it by providing an extension.
  • Provide your own extension and save the file
  • Copy the file to clipboard

Allowing the application to generate an extension

When you decode base64 contents, Quick Base64 can generate an extension for you file based on the mime-type of the decoded contents.
That is, Quick Base64 can decode base64 contents to it's original format without knowing the extension of the original file.
This is achieved using the file command. The file command can generate a mime type out of the decoded contents, then Quick Base64 will generate an extension based on the mime type.
This is really usefull if you have no idea about the contents you are decoding.

Providing your own extension

Also, you can also provide the extension for which the decoded file will be saved in.
This method comes in handy when you want to save the decoded contents in a specific file format.

Copying contents to clipboard

After decoding a file, you can just copy all of the contents to your system clipbord.
